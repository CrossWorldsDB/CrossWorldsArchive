How to Contribute to the Database

Overview

The SRCW DB is stored entirely on GitHub. All asset metadata is in data/assets.json. Asset files (textures, audio) live in the assets/ directory. Anyone with a GitHub account can contribute by submitting a Pull Request.

Note: Please only submit assets you have ripped yourself or that are already freely available to the community. Do not submit copyrighted material that has not been publicly extracted.

Steps to Add an Asset

  1. Fork the repository on GitHub by clicking the Fork button at the top right of the repo page.
  2. Clone your fork to your local machine:
    git clone https://github.com/CrossWorldsDB/CrossWorldsArchive.git
  3. Add your asset file to the correct subfolder inside assets/. For example, a character texture goes in assets/textures/characters/.
  4. Edit data/assets.json and add a new entry for your asset (see format below). Add it to the "assets" array.
  5. Commit your changes:
    git add .
git commit -m "Add: Sonic kart body texture v1.2.0"
git push origin main
  6. Open a Pull Request on GitHub from your fork to the main repository. Use a clear title describing what you added.

JSON Format

Each asset entry in data/assets.json must follow this structure:

{
  "id": "unique_snake_case_id",
  "name": "Human Readable Name",
  "description": "Short description of what this asset is and where it appears.",
  "type": "texture",
  "item_type": "character",
  "patch_version": "1.2.0",
  "category": "character",
  "file": "assets/textures/characters/filename.png",
  "resolution": "1024x1024",
  "tags": ["sonic", "character", "body"]
}

Field Reference:

idUnique snake_case identifier. Must not duplicate an existing ID.
nameDisplay name shown in the archive browser.
descriptionOne or two sentences about what the asset is.
typetexture or audio
item_typecharacter, vehicle, track, ui, effects, or misc
patch_versionGame patch version when this asset was ripped. Format: X.Y.Z
categorySame as item_type, used for internal grouping.
fileRelative path from repo root, e.g. assets/textures/ui/hud.png
resolutionImage resolution, e.g. 1024x1024. Use N/A for audio.
tagsArray of lowercase keyword strings for search.

Naming Conventions

  • IDs: character_sonic_body_diffuse, ui_hud_speedometer
  • Filenames: lowercase, underscores, no spaces: sonic_kart_body.png
  • Audio: include format in name if needed: bgm_splash_screen.ogg
  • Patch version: use the exact version shown in the game's main menu or patch notes

Pull Request Guidelines

  • One PR per batch of related assets (e.g. all textures from one patch update)
  • PR title format: Add: [category] [asset name(s)] [version]
  • Include a brief description of what you're adding and how it was extracted
  • Make sure all JSON is valid before submitting (use JSONLint ↗)
  • Verify the file paths in JSON match the actual file locations

JSON Template (Copy & Paste)

Use this template for a new asset entry:

{
  "id": "CHANGE_ME",
  "name": "CHANGE ME",
  "description": "CHANGE ME",
  "type": "texture",
  "item_type": "misc",
  "patch_version": "1.0.0",
  "category": "misc",
  "file": "assets/textures/misc/CHANGE_ME.png",
  "resolution": "512x512",
  "tags": ["tag1", "tag2"]
}

Remember to add a comma after the previous entry in the JSON array before pasting your new entry, or you will get a parse error.
