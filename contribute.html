How to Contribute

Overview

The archive is stored entirely on GitHub. Asset metadata lives in data/assets.json — generated automatically by the included Python script. Asset files go in the assets/ folder. Anyone with a GitHub account can contribute via pull request.

Only submit assets you ripped yourself or that are already publicly available in the community. Do not submit anything you don't have the right to share.

Folder structure

CrossWorldsArchive/
├── index.html
├── asset.html
├── contribute.html
├── css/style.css
├── js/main.js
├── data/
│   └── assets.json     
└── assets/
    ├── 01_Union/
    │   ├── UI/
    │   ├── Asset/
    │   ├── Map/
    │   └── Sound/
    ├── 02_Union/
    └── ...

The patch folder name (e.g. 01_Union) and the category folder directly inside it (e.g. UI, Asset) are read directly from the file path. Keep the same structure that Fmodel exports.

How to add assets

  1. Fork the repository on GitHub
  2. Clone your fork to your PC
  3. Copy your exported files into the correct folder under assets/
  4. Run python generate_assets.py from the repo root — this rewrites data/assets.json automatically
  5. Commit and push everything (the asset files + the updated JSON)
  6. Open a pull request back to the main repo

JSON format

The JSON is generated automatically by the script — you don't need to write it by hand. Each entry looks like this: 

{
  "file": "assets/01_Union/UI/Common/UI_Common_Icon.png",
  "type": "texture",
  "patch": "01_Union",
  "category": "UI"
}
filePath from repo root to the asset file
typetexture or audio
patchPatch folder name, e.g. 01_Union
categoryTop-level folder inside the patch, e.g. UI, Asset, Map

Guidelines

  • Keep the folder structure matching what Fmodel exports — don't rename folders
  • Use lowercase filenames with no spaces if you rename anything
  • One pull request per patch update ideally, not one PR per file
  • PR title format: Add: 02_Union UI textures
  • Check your JSON is valid before submitting at jsonlint.com
